```
<type>(<scope>): <описание на русском>

[опциональное тело]

[опциональные footer]
```

---

## 🎨 Эмодзи в коммитах

Для визуального выделения секций используй следующие эмодзи:

| Эмодзи | Секция |
|--------|--------|
| ✨ | Новые возможности / feat |
| 🔧 | Изменения / refactor |
| 🐛 | Исправление багов / fix |
| 🧪 | Тесты / test |
| 📝 | Документация / docs |
| 💾 | Обратная совместимость |
| ⚡ | Оптимизация / perf |
| 🚀 | Запуск / релиз |
| 🔒 | Безопасность |
| 🗑️ | Удаление кода |
| ♻️ | Рефакторинг |

## 📌 Формат коммитов (GitHub Desktop)

**Summary (заголовок):**
```
<type>(<scope>): <краткое описание (до 50 символов)>
```

**Description (тело коммита):**
```
✨ Новые возможности:
- Пункт 1
- Пункт 2

🔧 Изменения:
- Пункт 1
- Пункт 2

🧪 Тесты:
- Пункт 1

💾 Обратная совместимость:
- Пункт 1
```

## 📌 Пример коммита

```bash
feat(v1.6.0): дедупликация доменов и дефолтный профиль

✨ Новые возможности:
- Дедупликация доменов: проверка на дубликаты (регистронезависимо)
- Нормализация ввода: удаление пробелов, http://, https://, www., завершающего слеша
- Дефолтный профиль для триггеров: запоминание и восстановление профиля
- ComboBox в ServicePage для выбора дефолтного профиля
- CustomDialog уведомление при дубликате

🔧 Изменения:
- SettingsService: добавлено поле DefaultProfileFileName (string?) в AppSettings
- MainViewModel: реализован NormalizeDomainInput(), обновлены Add/RemoveCustomDomain
- MainViewModel: добавлено ObservableProperty defaultProfileFileName
- MainViewModel.Orchestrator: логика запоминания/восстановления профилей
- ServicePage.xaml: добавлена секция ДЕФОЛТНЫЙ ПРОФИЛЬ

🧪 Тесты:
- Создан DomainDeduplicationTests.cs: 32 теста
- ✅ 129/129 тестов пройдены

💾 Обратная совместимость:
- DefaultProfileFileName инициализируется как null
- Fallback на SelectedProfile если DefaultProfileFileName не задан
- Безопасный fallback если профиль удалён
```

## 📋 Типы

| Тип        | Когда использовать                      |
| ---------- | --------------------------------------- |
| `feat`     | Новая функциональность                  |
| `fix`      | Исправление бага                        |
| `docs`     | Документация                            |
| `style`    | Форматирование, пробелы                 |
| `refactor` | Рефакторинг без изменения поведения     |
| `perf`     | Оптимизация производительности          |
| `test`     | Добавление тестов                       |
| `build`    | Изменения в системе сборки              |
| `ci`       | CI/CD конфигурация                      |
| `chore`    | Мелкие задачи (обновление зависимостей) |

## 📌 Примеры

```bash
feat(core): добавлен сервис загрузки конфигурации
fix(ui): исправлен лаг при загрузке списка
perf(ai): оптимизирован Thompson Sampling на 40%
test(services): добавлены тесты для DataService
docs: обновлён README с инструкциями
refactor(core): вынес валидацию в отдельный класс
```
